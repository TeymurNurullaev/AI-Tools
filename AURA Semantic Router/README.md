# 📑 Спецификация протокола AURA Semantic Router (v4.0)

### Распределенное семантическое внимание и ролевая маршрутизация контекста для ИИ-агентов

---

## 🧭 1. Введение и назначение протокола

Современные большие языковые модели (LLM) обладают огромными контекстными окнами, но при работе со сложными программными системами они сталкиваются с фундаментальной проблемой: **семантическим шумом**.

Если загрузить в ИИ весь репозиторий, документацию и историю задач разом, модель начинает путать уровни абстракции, тратить лишние токены и галлюцинировать. Она видит отдельные файлы, но теряет понимание цифрового организма в целом.

**AURA Semantic Router** — это семантический протокол распределения внимания (Semantic Attention Distribution Protocol). Он разработан для того, чтобы автоматически разделять знания о системе на изолированные слои и динамически собирать для каждого ИИ-агента **только тот контекст, который необходим для его роли и текущей задачи**.

---

## 🎯 2. Какие проблемы решает протокол

* **Выжигание контекста и бюджетов:** Вместо передачи сотен килобайт избыточного кода, ИИ получает точечный слепок системы объемом в несколько килобайт.
* **Галлюцинации моделей:** ИИ галлюцинирует, когда данные смешиваются в хаотичную кучу. Протокол изолирует кодовую базу от верхнеуровневой философии, загоняя ИИ в жесткие рамки спецификаций.
* **Смешение когнитивных режимов:** Протокол активирует у ИИ нужные синаптические связи. Агент-программист получает только инструкции к коду (не отвлекаясь на глобальный бизнес-стратегию), а агент-архитектор — карту связей (не отвлекаясь на синтаксис функций).
* **Сложный онбординг новых ИИ-сотрудников:** При подключении к проекту новому ИИ-агенту достаточно сделать один семантический запрос, чтобы мгновенно понять правила игры в незнакомом репозитории.

## 🏗️ 3. Схема работы семантического роутера

Протокол разделяет знания любого узла системы (папки или модуля) на **4 независимых семантических слоя**:

### Жизненный цикл обработки задачи:

```
    Входящая задача (Сделай рефакторинг / Проверь безопасность)
                          │
                          ▼
               ┌─────────────────────┐
               │   Semantic Router   │
               └─────────────────────┘
              (Определяет Роль и Слой)
                          │
         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
   [ Кодер ]         [ Стратег ]      [ Безопасник ]
  (organ.md)        (manifest.md)    (guardrails.md)
         │                │                │
         └────────────────┼────────────────┘
                          ▼
            ┌───────────────────────────┐
            │  Context Assembly Engine  │
            └───────────────────────────┘
             (Сборка финального пакета)
                          │
                          ▼
                 Выполнение в LLM

```

## 🗂️ 4. Стандарт структуры файлов (Абстрактные примеры)

Для демонстрации протокола миру мы используем **абстрактный пример** компонента авторизации, чтобы стандарт подходил для любого фреймворка, языка программирования (PHP, Python, Node.js) или архитектуры.

### Слой 1: Системная ДНК (`manifest.md`)

* **Для кого:** Архитекторы, Оркестраторы, Диспетчеры Сети.
* **Суть:** Где узел находится в системе, с чем связан и какая у него критичность.

```yaml
# manifest.md
identity:
  node: "/services/auth_provider"
  type: "system_core"
  subsystem: "Security & Gateways"
  version: "2026.1.0"
  criticality: "mission_critical"

architecture_relations:
  data_stores:
    - target: "users_credential_db"
      access: "read_only"
      purpose: "Проверка хэшей паролей и токенов доступа."
  
  external_connections:
    - service: "OAuth2_Provider"
      protocol: "HTTPS/REST"
      purpose: "Валидация внешних сессий пользователей."

dependencies:
  upstream:
    - node: "/gateway/ingress_router"
      type: "hard"

```

---

### Слой 2: Паспорт органа (`organ.md`)

* **Для кого:** Кодеры, Разработчики, Агенты Рефакторинга.
* **Суть:** Как устроен код внутри, какие методы вызывать, какие локальные правила кодстайла соблюдать.

```yaml
# organ.md
organ_identity:
  node: "/services/auth_provider"
  engine_pattern: "Stateless_Service"
  strict_typing: true

runtime_interfaces:
  methods:
    - name: "validateSession"
      access: "public"
      input_format: "JWT_String"
      output_format: "Boolean"
      purpose: "Проверка валидности текущей сессии в рантайме."
    
    - name: "issueToken"
      access: "private"
      input_format: "User_Entity_Object"
      output_format: "JSON_Web_Token"
      purpose: "Генерация нового токена доступа после успешной авторизации."

coding_guardrails:
  allowed_libraries: ["native_crypto", "jwt_helper"]
  prohibited_patterns:
    - "Запрещено использовать внешние тяжелые ORM в этом узле. Писать только нативные оптимизированные запросы."
    - "Запрещено хранить промежуточные состояния внутри класса (класс должен оставаться Stateless)."

```

### Слой 3: Контур безопасности и ограничений (`guardrails.md`)

* **Для кого:** Безопасники, Тестировщики, DevOps-инженеры.
* **Суть:** Какие данные защищать, какие тесты гонять, какие лимиты производительности установлены.

```yaml
# guardrails.md
security_policies:
  access_level: "isolated"
  encryption_standard: "AES-256-GCM"

data_sensitivity:
  contains_personal_data: true       # Паспорта, имена, почты
  contains_credentials: true         # Хэши паролей и сессионные ключи
  anonymization_required: true       # Логи рантайма должны принудительно маскировать персональные данные
  public_exposure_allowed: false     # Запрещено отдавать структуру узла наружу при публичных сканированиях

test_contracts:
  required_suites:
    - "token_expiration_regression_test"
    - "crypto_signature_integrity"
  blast_radius:
    - "/gateway/ingress_router"
  failure_policy: "block_deployment"

```

## ⚡ 5. Концепция сборщика ядра (Context Assembly Engine)

Спецификация работает в связке с автоматизированным скриптом `SemanticRouter.py`, который запускается на сервере или в CI/CD пайплайне. Его задача — сделать обход директорий репозитория и выдать на выходе сжатый текстовый файл семантического слоя для отправки в LLM.

### Интерфейс вызова скрипта (Примеры команд):

1. **Сборка только глобальной ДНК системы:**
```bash
python SemanticRouter.py aurav3

```


*Результат:* Скрипт обходит все папки, вытаскивает исключительно файлы `manifest.md` и собирает верхнеуровневую карту проекта для Агента-Стратега. Лишний код и инструкции не трогаются.
2. **Сборка полной базы знаний репозитория:**
```bash
python SemanticRouter.py all

```


*Результат:* Полный слепок всех четырех слоев данных по всем узлам. Используется для глобального резервного копирования смысловой памяти или инициализации центрального ИИ-Оркестратора.
3. **Сборка только опыта и рецептов:**
```bash
python SemanticRouter.py playbook

```


*Результат:* Из всех папок вытаскиваются только файлы `playbook.md`. Идеальный контекст для Агента-Оптимизатора, который ищет слабые места в процессах разработки на основе прошлых ошибок команды.
4. **Точечная сборка под задачу (Пример расширенного использования):**
```bash
python SemanticRouter.py node=/services/auth_provider role=Coder

```


*Результат:* Скрипт выдаст только `manifest.md` (где я?) и `organ.md` (как писать код?) для конкретного узла, обеспечивая снайперски точный контекст для ИИ-программиста.
