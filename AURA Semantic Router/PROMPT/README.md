# 🛸 AURA Semantic Engine Prompts (v4.3)

Репозиторий содержит ультимативные **мета-промпты (генераторы слоев)** для автоматического развертывания и документирования семантической архитектуры **AURA v3.1 / v4.3** на базе ИИ.

Эти промпты превращают рутинную паспортизацию кода в конвейер. Они переводят архитектурное описание из разряда «забытой документации» в формат **Исполняемого Контекста (Context as Code)**, который мгновенно понимают как люди, так и автономные ИИ-агенты через API.

---

## 💡 Зачем это нужно?

В сложных распределенных системах (таких как экосистема `BDGSite AI`) контекст постоянно размывается, а у ИИ-моделей при чтении всего исходного кода перегружается контекстное окно (Prompt Injection / Noise). 

Мы разделили реальность каждого модуля (узла) системы на **4 независимых слоя внимания**:
1. **Слой 1: Системная ДНК (`manifest.md`)** — связи, зависимости, критичность узла.
2. **Слой 2: Паспорт Органа (`organ.md`)** — интерфейсы методов, разрешенные и запрещенные коллекции.
3. **Слой 3: Контур Безопасности (`guardrails.md`)** — чувствительность данных, лимиты, радиус поражения (`blast_radius`).
4. **Слой 4: Журнал Опыта (`playbook.md`)** — извлеченные из логов уроки, баги, антипаттерны.

**Проблема:** Писать эти YAML-манифесты вручную для каждого класса — адский ручной труд.
**Решение:** Эти промпты полностью автоматизируют процесс. Вы даете ИИ сырой код, он возвращает идеальный семантический слой за 3 секунды.

---

## 🛠️ Как это работает? Конвейер из 3 шагов

Процесс интеграции любого нового модуля или класса в протокол автоматизирован до предела. 

### Шаг 1: Написание кода
Вы создаете новый модуль, класс (например, `Assembly.php`) или пишите техническое задание для нового микросервиса.

### Шаг 2: Делегация ИИ
Вы берете целевой мета-промпт из этого репозитория (например, для Слоя 2 — `Organ Generator`), прикрепляете к нему ваш сырой код и отправляете любой современной LLM (через веб-интерфейс или напрямую через API в вашем CI/CD контуре).


[Ваш Мета-Промпт из репозитория]
+
[Сырой код вашего модуля / класса / ТЗ]


### Шаг 3: Деплой спецификации
ИИ мгновенно анализирует код, вытаскивает структуру, проверяет правила и выдает **идеально валидный YAML/Markdown блок**. Вам остается только сохранить его в папку модуля под нужным именем (`manifest.md`, `organ.md` и т.д.). 

`SemanticRouter.py` мгновенно подхватит новый слой и включит его в общую топологию распределения семантического внимания.

## 🧩 Анатомия репозитория

* `PROMPT_LAYER_1_SYSTEM_DNA.md` — Генератор ДНК узла. Связывает архитектурные узлы, базы данных и внешние API (например, Qdrant).
* `PROMPT_LAYER_2_ORGAN_SPECS.md` — Снайперский разбор методов класса. Внедряет жесткие ограничения на кодинг (запрет тяжелых библиотек, контроль Stateless состояний).
* `PROMPT_LAYER_3_GUARDRAILS.md` — Настройка РЭБ-контура безопасности. Автоматически размечает персональные данные, уровни доступа и сценарии блокировки деплоя.
* `PROMPT_LAYER_4_PLAYBOOK.md` — Живая память компонента. Сюда ИИ дописывает исторические ошибки и антипаттерны, предотвращая наступление на одни и те же грабли в будущем.

## 🤖 API и Автономное использование (AI Net)

Эти промпты оптимизированы для использования в **Multi-Agent Swarms** (микроагентных сетях). Вы можете настроить воркер так, чтобы при каждом коммите или падении тестов в рантайме, система автоматически обращалась к этим промптам, обновляла семантические слои и сама накапливала опыт в `playbook.md` без участия человека.
