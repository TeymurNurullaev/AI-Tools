# AITools / AURA NexusCompiler

### Описание проекта

AURA NexusCompiler — это системный инструмент для автоматического создания семантической топологии и распределенной памяти больших программных комплексов. Скрипт сканирует целевую директорию проекта, очищает граф от технического шума и собирает распределенные по папкам файлы описаний (`manifest.md`) в единую инженерную спецификацию. Полученный документ служит универсальным контекстным мостом, который позволяет любой современной LLM (OpenAI, Claude, YandexGPT) мгновенно понять архитектуру, бизнес-логику и межсистемные связи проекта без загрузки сырого исходного кода.

### Решаемые проблемы и целевая аудитория

* **Контекстное удушье ИИ:** Большие проекты не помещаются в контекстное окно моделей или забивают его шумом (библиотеки, стили). Инструмент сжимает архитектуру до чистых смыслов, экономя до 90% полезного объема памяти ИИ.
* **Галлюцинации моделей:** ИИ часто ошибается в назначении изолированных модулей. Скрипт поставляет жесткий контур правды, исключая неверную интерпретацию кода.
* **Устаревание документации:** Централизованные ТЗ умирают по мере роста системы. Данный подход распределяет документацию по папкам («ДНК-паспорта»), делая ее актуализацию частью процесса разработки.

**Кому помогает:**

* **ИИ-Агентам:** для быстрой и безошибочной онбординг-инициализации в кодовой базе.
* **Системным Архитекторам:** для контроля геометрии, связей и топологии разрастающейся системы.
* **Разработчикам:** для мгновенного понимания логики незнакомых модулей через локальные манифесты.

### Выходная структура данных (3 Блока)

Результатом работы является один монолитный Markdown-файл, разделенный на три логических слоя:

1. **System Overview:** Верхнеуровневое описание всей системы, извлеченное из корневого манифеста.
2. **Directory Topology:** Кристально чистый скелет проекта в виде псевдографического дерева без мусорных файлов.
3. **Component Manifests:** Реестр всех найденных локальных паспортов папок с четкими семантическими границами (`Context Boundary`).

### Спецификация формата `manifest.md`

Каждая папка, требующая описания для ИИ, снабжается легковесным файлом `manifest.md` следующего формата:

```markdown
---
node: "/Public/Module/AI"
type: "core_module"
subsystem: "BDGSite AI Engine"
db_relations:
  - table: "ai_sessions"
    type: "write/read"
  - table: "company_registry"
    type: "read_only"
external_connections:
  - system: "Qdrant Vector DB"
    protocol: "gRPC"
---
```

### Настройка и Конфигурация

Конфигурация рантайма осуществляется внутри блока `БЛОК РУЧНОЙ НАСТРОЙКИ АРХИТЕКТУРЫ` самого скрипта:

* `TARGET_DIR`: Абсолютный путь к сканируемой директории проекта.
* `OUTPUT_MARKDOWN_FILE`: Абсолютный путь к финальному файлу отчета.
* `IGNORE_DEEP_SCAN`: Список путей для точечной блокировки сканирования. Вариант без `*` блокирует проход полностью. Вариант с `/*` показывает только первый уровень вложений.
* `EXCLUDE_DIR_NAMES_STR`: Разделенный точкой с запятой список имен папок, которые нужно отобразить в дереве, но не сканировать их содержимое (`css;js;lib;.git`).
* `EXCLUDE_EXTENSIONS_STR`: Список расширений файлов, полностью исключаемых из финального отчета.
* `COMMENTS_MAP`: Словарь для жесткой привязки базовых комментариев к ключевым узлам дерева.

### Инструкция по использованию

1. Разместите файл `manifest.md` в корневом каталоге проекта и аналогичные файлы во вложенных модулях, требующих индексации.
2. Откройте `NexusCompiler.py` и заполните блок конфигурации под параметры вашего сервера.
3. Запустите компилятор из терминала:
```bash
python3 NexusCompiler.py

```

4. Сгенерированный файл `directory_structure.md` передайте ИИ-модели в качестве стартового контекста перед началом работы над проектом.
